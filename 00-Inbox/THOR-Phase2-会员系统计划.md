# THOR 会员系统 + 项目管理 + 分级权限
> Phase 2 开发计划 | 2026-06-18

---

## 整体架构

```
┌──────────────────────────────────────────────┐
│  前端（HTML + CSS + JS）                      │
│                                               │
│  报价单页面                                    │
│    ├── 保存项目 → localStorage / Supabase      │
│    ├── 加载项目 ← 项目列表                     │
│    └── 根据 role 显示/隐藏功能                  │
│                                               │
│  项目看板（dashboard.html）                     │
│    ├── 项目列表（搜索/筛选/排序）               │
│    ├── 统计概览（本月报价数/成交数/金额）        │
│    └── 按门店/人员查看                          │
│                                               │
│  登录页（login.html）                          │
│    ├── 账号密码登录                            │
│    └── 角色选择（L1~L4 展示不同权限）           │
└──────────────────────────────────────────────┘

后端（Phase 2b - Supabase）
  ├── projects 表
  ├── users 表 + Auth
  └── API 接口
```

## Phase 2a：前端本地版（先做这个）

### 1. 项目存取（localStorage）
```
报价单页面:
  ├── "保存" 按钮 → 存到 localStorage
  │    ├── 项目名/客户名 + 当前时间
  │    └── 所有表格数据序列化为 JSON
  │
  ├── "加载" 按钮 → 打开项目列表
  │    ├── 从 localStorage 读取所有项目
  │    ├── 显示列表（客户名/日期/金额/状态）
  │    └── 点击 → 恢复数据到表单
  │
  └── "删除" 项目
```

### 2. 数据格式
```json
{
  "id": "THOR-20260618-001",
  "name": "张先生厨房",
  "customer": {"name":"张先生","phone":"138...","address":"东莞..."},
  "items": {
    "cabinet": [...],
    "countertop": [...],
    "hardware": [...],
    "sink": [...], 
    "electric": [...],
    "extra": [...]
  },
  "totals": { "subtotal": 12907, "grandTotal": 12907 },
  "status": "draft|quoted|confirmed|ordered",
  "salesperson": "张三",
  "store": "门店A",
  "createdAt": "2026-06-18T08:00:00Z",
  "updatedAt": "2026-06-18T08:00:00Z"
}
```

### 3. 项目看板（dashboard.html）
```
├── 顶部: 统计卡片（本月报价数/总金额/平均单值）
├── 中间: 项目列表表格
│    ├── 项目名 | 客户 | 金额 | 状态 | 日期 | 操作
│    └── 搜索/筛选/排序
└── 底部: 分页
```

### 4. 分级权限（前端模拟）
```
L1 - 销售：只能看自己的报价
L2 - 主管：看全部报价 + 看出厂价
L3 - 老板：全部权限 + 管理用户
L4 - 超级管理员：系统设置

当前实现：
  ├── 模拟登录选择角色
  ├── 界面根据 role 隐藏/显示元素
  └── localStorage 存当前登录信息
```
